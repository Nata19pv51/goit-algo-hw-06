В результаті виконання DFS і BFS алгоритмів зі стартовою точкою в Києві і кінцевою - Біла Церква отримано різні обходи і шляхи:
DFS обхід графа і шлях: Київ -> Глеваха -> Фастів -> Біла Церква. 

BFS обхід графа: Київ -> Глеваха -> Васильків -> Обухів -> Фастів -> Біла Церква. Шлях: Київ -> Глеваха -> Біла Церква

Різниця в отриманих шляхах пояснюється фундаментальним принципом роботи кожного алгоритму. 
--  Для DFS це пошук у глибину, тобто йде вглиб до кінця поточної гілки, перш ніж повертатися і досліджувати інші гілки (Київ, Глеваха, Фастів, Біла Церква). DFS не шукає найкоротший шлях, він просто знаходить будь-який шлях до цілі, і не повертається, щоб перевірити інші, доки не знайде ціль.
--  Для BFS це пошук у ширину, тобто обходить рівень за рівнем, спочатку досліджуючи всіх прямих сусідів (Глеваха, Васильків, Обухів), потім їхніх сусідів (Фастів) і т.д. BFS може знайти найкоротший шлях за кількістю ребер
